# Mexico CSAT League 🇲🇽

A live CSAT leaderboard for the Mexico TSE team — **Team Norma** vs **Team Pablo**.  
Hosted on GitHub Pages at [normarsdd.github.io/mexico-csat-board](https://normarsdd.github.io/mexico-csat-board).

---

## How it works

The dashboard reads from `data.js`, which is auto-generated by querying Snowflake.  
Scoring: **CSAT with customer comment = 2 pts** / **CSAT without comment = 1 pt**.

---

## Updating the data

### Prerequisites

- `snow` CLI installed and authenticated (`norma.ruysanchez@datadoghq.com`)
- `git` configured with access to `normarsdd/mexico-csat-board`
- `gh` CLI with both accounts set up: `normarsdd` and `norma-ruysanchez_ddog`

### Option A — Single command (recommended)

Just say **"run the csat league"** to Claude Code and it will run the full update flow automatically.

### Option B — Run it manually

```bash
bash ~/DDRepos/mexico-csat-board/update_csat_board.sh
```

This will:
1. Query Snowflake for all May 2026 CSAT data (both teams)
2. Regenerate `data.js` with updated scores and history
3. Auto-commit and push `data.js` to `origin` (normarsdd/mexico-csat-board → GitHub Pages)

---

## Pushing to ddoghq-sandbox

The dashboard also lives at the Datadog sandbox org. After running the update script, push to the second remote:

```bash
gh auth switch -u norma-ruysanchez_ddog
git -c credential.helper='!gh auth git-credential' push ddog main
gh auth switch -u normarsdd
```

Or say **"run the csat league"** to Claude Code — it does all of this in one shot.

---

## Pushing design/code changes

After editing `index.html`, commit and push to both remotes:

```bash
# Push to GitHub Pages (personal account)
git add index.html
git commit -m "your message"
git push origin main

# Push to ddoghq-sandbox (work account)
gh auth switch -u norma-ruysanchez_ddog
git -c credential.helper='!gh auth git-credential' push ddog main
gh auth switch -u normarsdd
```
